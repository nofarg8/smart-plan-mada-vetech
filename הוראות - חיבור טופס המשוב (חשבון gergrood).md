# חיבור טופס המשוב ודיווח התקלות - הוראות לנופר

הכפתור הצף "משוב ותקלות" כבר בנוי באתר, אבל הוא **מוסתר** עד שנחבר אותו.
החיבור: סקריפט קטן שרץ **בחשבון האישי שלך (gergrood@gmail.com)** - הדיווחים
ייאספו בגיליון בדרייב האישי שלך, ותקלות ישלחו לך גם מייל התראה מיידי.

## מה עושים (5 דקות, פעם אחת)

1. נכנסים ל-script.google.com **כשמחוברת לחשבון gergrood@gmail.com** (לא ההתיישבותי!).
2. לוחצים **New project** (פרויקט חדש).
3. מוחקים את הקוד הריק שבעורך, ומדביקים במקומו את **כל** התוכן של הקובץ
   `apps-script/feedback.gs` (מהתיקייה של הפרויקט).
4. בודקים עם Ctrl+F שהמילה `feedback` מופיעה בקוד שהודבק.
5. שומרים (Ctrl+S), נותנים שם לפרויקט, למשל: "משוב תומכת הוראה".
5א. **אישור הרשאות (חובה לפני הפרסום!):** בסרגל למעלה בוחרים בתיבת הפונקציות
   את **authorizeNow** ולוחצים **Run** (הפעלה). גוגל תבקש אישור - מאשרים
   (אם מופיע "Google hasn't verified this app": לוחצים Advanced ואז
   "Go to ... (unsafe)"). ההרצה נוגעת בדרייב, בגיליונות ובמייל כדי שכל
   ההרשאות ייתפסו. רק אחרי האישור ממשיכים לפרסום.
6. לוחצים **Deploy -> New deployment -> Web app**, ומגדירים:
   - Execute as: **Me**
   - Who has access: **Anyone**
7. מאשרים את מסך ההרשאות אם מופיע (גישה לדרייב ולמייל שלך).
8. מעתיקים את כתובת ה-URL שמתקבלת ושולחים לי (Claude) - אני מעדכן אותה
   בקוד האתר, והכפתור יופיע לכולן.

## מה יקרה אחרי החיבור

- בדרייב האישי שלך ייווצר גיליון בשם **"משוב ודיווח תקלות - תומכת הוראה אישית"**
  עם שתי לשוניות: "משוב" ו"דיווח תקלות". כל שליחה = שורה חדשה.
- על כל **דיווח תקלה** תקבלי **מייל מיידי** עם כל הפרטים (כולל נייד של המדווחת
  ופרטי הדפדפן שלה - לאבחון).
- משוב רגיל נאסף בשקט בגיליון, בלי מייל.

## תזכורת לעדכונים עתידיים

כמו במסירה: אם נשנה את הקוד בעתיד - תמיד **Deploy -> New deployment**
(לא New version), ולשלוח לי את ה-URL החדש.
