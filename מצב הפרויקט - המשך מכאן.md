## הזרימה שעובדת עכשיו (מקצה לקצה, חי)
1. **מסך פתיחה / הזדהות** (`LoginScreen` ב-`App.tsx`): שם, אימייל, סמל מוסד, אימייל רכז. **אימות:** שולף מהסטטוס החי לפי סמל מוסד; אם אימייל הרכז תואם -> נכנס.
   - **כניסת דיבאג:** כפתור "כניסת דיבאג" (או סמל מוסד `DB`) מתחבר לסטטוס האמיתי של נופר (סמל **999999** - "נופר דיבאג", ז'=11/ח'=12, יוזמות "עין העדשה"). מאחורי `DEBUG_ENTRY` ב-`App.tsx` - **לכבות ל-false לפני פרסום.**
2. **פאנל "פרטי בית הספר"** (`SchoolInfo`): מציג מה שנשלף מהסטטוס - ש"ש לשכבה, מעבדה, יריד. **לא שואלים את המורה מה שכבר בסטטוס.**
3. **מערכת שעות** (`ScheduleEditor`): מולאה מראש מש"ש הסטטוס; המורה מחלקת לימים. `weeklyHours` = הסכום.
4. **פריסה שבועית אישית** (`WeeklyPlan`/`WeekRow`): לכל שבוע, השיעורים של המורה (ב'/ג'/ה'...) ממולאים: נושא ללימוד, ומתי בדיוק משימת מודל (עם סוג+פירוט+תג "אירוע הערכה"), חקר או מבחן. + מבט חודשי קומפקטי למעלה.

## המנוע (`app/src/engine/plan.ts`) - הפונקציות
- **`buildPlan(input)`** -> `Plan`. `input = { grade, weeklyHours, modelTaskDays? }`.
- **`scheduleTopics`** - פורס נושאים על **כל השנה** לפי **קצב** (`pace = סך שעות / סך מקדמי שבועות`, מוגבל ל-weeklyContent). לא דוחס לתחילת השנה -> תוכן בכל חודש ספט-יוני.
- **`scheduleModelTasks`** - משבץ משימות מודל **לפי הנושא המשויך** (`topicNames`), מפוזרות לשבוע הפחות-עמוס (לא נערמות). fallback: חודש -> מחרוזת.
- **`researchTrack`** - אבני דרך חקר בתאריכים **הרשמיים**: 18.1 הגשת נושא, 1.2/15.2 פוסטרים, **יריד מחוזי 16.3**, 2.5 הגשה לכנס ארצי, **יריד ארצי 1.6**.
- **`buildWeeklySchedule(plan, teacherSlots)`** -> `WeekSchedule[]` - **הלב.** לכל שבוע לימוד: משבצות ההוראה של המורה, מלאות בנושא + משימות מודל/חקר/מבחן בשיעורים המתאימים (עם `taskType`, `detail`, `isAssessment`). **מקיף:** מחלק את *כל* הנושאים הפעילים על השיעורים (בלי לדלג), ומצרף `topicList` + `subItems` (הרחבה/רשות). **חג:** שיעור שנופל עליו חג רשמי (`officialHolidayOn`) מסומן `kind:'חג'` ולא מקבל תוכן.
- **`droppedTopics` (ב-`EngineInput`)** - נושאים שהמורה בחרה לצמצם במחסור; יורדים מהפריסה. `plan.capacityHours/plannedCoreHours/shortfallHours` מזינים את `AdjustPanel`.
- **`buildMonthlyCalendar(plan)`** -> מבט חודשי: שעות, נושאים, חגים נקיים, מבחנים, משימות מודל, יוזמות STEM, חקר.
- `buildPlanTable`/`crossYearModelTasks` - קיימים, **לא מוצגים** (הוחלפו בפריסה השבועית). אפשר להסיר בעתיד.

## מקורות חיים (Live) - שניהם מחוברים ✅

"לשלוח החוצה" מותר, "למשוך פנימה אנונימית" חסום. אם צריך לחבר עוד גיליון פרטי - **Publish to web CSV, לא Apps Script.** (הקובץ `apps-script/status.gs` נשאר כגיבוי לשיטה החלופית, לא בשימוש.)
   - **פרטיות:** הלשונית המסוננת בלי טלפונים/שדות מנהליים; כן מייל רכז (נחוץ לאימות + קלנדר) -> בקישור ציבורי, בידיעת נופר.

## 📦 פאזת המסירה (בנויה ועובדת - קרא לפני שנוגעים בה)
**זרימה:** כפתור אחד ב-Header, **"הורידי PDF וקבלי גוגל קלנדר"** -> `ResultScreen.doFinalize` -> מוסיף class `exporting` על הדף (מסתיר פקדים, כופה פתיחת כל תתי-הנושא ל-PDF) -> קורא ל-`finalizePlan` ב-`app/src/export.ts`.

**`app/src/export.ts` (כל לוגיקת הייצוא):**
- `generatePdfBase64(element)` - יוצר PDF מהדף עם **html2pdf.js** (הותקן: `npm i html2pdf.js`; יש `src/html2pdf.d.ts` להצהרת טיפוס). מחזיר base64.
- `buildICS(plan, session, weekly, stamp)` - בונה מחרוזת .ics תקינה (RFC 5545, כולל קיפול שורות ל-Hebrew). `planEvents(...)` מחזיר את אותם אירועים כרשימת `{start,end,title}` (תאריכים מקומיים YYYY-MM-DD) - **זה מה שנשלח לסקריפט לבניית היומן.**
- **תוכן הקלנדר (~193 אירועים):** אירוע לכל **יום הוראה** עם מה שמלמדים (מ-`weekly` slots, kind 'נושא', תאריך `sl.dateISO`), + משימות מודל/חקר/מבחנים (מ-`plan.deadlines`), + יוזמות, + חופשות רשמיות. `SlotAssignment.dateISO` מחושב במנוע ב-`buildWeeklySchedule` (יום שלא נופל בשבוע חלקי -> null -> לא נכנס, מונע כפילות בשבוע הראשון).
- `finalizePlan(...)` - מייצר PDF, מוריד אותו למורה, ו-`postDelivery` שולח POST ל-`DELIVERY_URL` (Content-Type text/plain כדי לעקוף preflight, כמו בסטטוס). ה-payload: `{action:'deliverPlan', schoolId, schoolName, teacherName, teacherEmail, coordinatorEmail, calendarName, pdfBase64, icsContent, events}`.
- `DeliverResult.calendarShared` - נגזר מ-`data.calendar && !data.calendarError`; ה-UI מציג הודעה כנה (אם היומן נכשל -> "היומן עדיין לא הופעל", לא "שותף").

**`apps-script/delivery.gs` (רץ בחשבון ההתיישבותי, Web App):**
- `doPost` -> `deliverPlan_`: תת-תיקייה `שם ביה"ס - סמל` בתוך `DELIVERY_PARENT_ID` -> שומר PDF + .ics (דורס לפי שם). `syncCalendar_` יוצר/מוצא יומן לפי שם, **מנקה אירועים בטווח השנה ובונה מחדש** (עדכון בלי כפילות), ומשתף (`shareCalendar_` -> `Calendar.Acl.insert` reader) עם המורה + הרכז. `MailApp` שולח .ics לרכז.
- **דורש:** שירות מתקדם "Google Calendar API" (Services) + הרשאת יומן מאושרת (ראה "מה שלמדנו על הרשאות" למעלה).
- כל חלק ב-try/catch -> תקלה ביומן/מייל לא חוסמת את תיוק ה-PDF.

**מגבלות/פתוחים במסירה (ראה גם "פתוחים"):**
- **מייל למורה מהסטטוס בדיבאג:** בכניסת דיבאג `teacherEmail = school.coordinatorEmail` (שניהם מהסטטוס) - לכן טעות במייל בסטטוס פגעה גם ביומן וגם ברכז. בזרימה אמיתית המורה מקלידה מייל בעצמה.
- **ריבוי יומנים:** נוצר יומן לכל מורה בחשבון ההתיישבותי - עם מאות מורים יצטבר. לשקול.
- **זמן:** יצירת ~193 אירועי יומן לוקחת ~40-80 שניות ("מפיק..." ארוך). תקין, אפשר לייעל בעתיד.
- **PDF לרכז:** כרגע הרכז מקבל **רק .ics** במייל; ה-PDF אצלו דרך תיקיית ה-Drive. נופר ביקשה לשקול לצרף PDF גם למייל (שינוי קטן ב-`delivery.gs` + פרסום מחדש).

- **הערך = הפריסה האישית לפי שעות המורה, לא טבלאות.** נופר התסכלה כשקיבלה טבלאות. תמיד לחשוב "מה המורה צריכה לעשות בשיעור הזה".
- **בלי טקסט פיתוח ב-UI. לעולם.** לסרוק לפני סיום.
- **חיבור גיליון פרטי = Publish to web CSV** (לא Apps Script, שנחסם בארגון). לשונית מסוננת עם QUERY לפרטיות.
- **הגאנט + הסטטוס חיים** - לא להטמיע כעותק. הם מתעדכנים.
- כשנופר נותנת מקור (PDF/תמונה/גיליון) - **לקרוא אותו** (pypdf זמין; docx/xlsx = zip של XML; תמונה = לקרוא ישירות). לא לנחש.
- כשמקורות רשמיים סותרים - **להציף לנופר**, לא להכריע לבד.
- **פאזת המסירה - לקחים:**
  - נופר **לא-טכנולוגית ומתלחצת מצעדים טכניים** - להסביר בפשטות מוחלטת, "מי עושה מה", ולהדריך צעד-צעד. לא להציף בז'רגון (base64/URL/ספריות).
  - **POST ל-Apps Script Web App כן עובד בארגון** (בניגוד למה שחשבנו על הסטטוס - שם החסימה הייתה על doGet אנונימי; המסירה עובדת דרך doPost). הדפוס: PDF נוצר בצד-לקוח, נשלח base64, הסקריפט שומר - בדיוק כמו `savePdfSummary_` של הסטטוס.
  - **הרשאות יומן ב-Apps Script הן מלכודת:** צריך Services→Calendar API + הרצת פונקציה שנוגעת ב-CalendarApp לאישור + New deployment אחרי האישור. "no permissions popped up" בהרצת doGet = doGet לא נוגע ביומן; צריך `authorizeNow` שכן נוגע.
  - **בעיות "לא הגיע" הן כמעט תמיד נתונים, לא קוד** - טעות o כפול במייל בסטטוס פגעה גם ביומן וגם ברכז. תמיד לבדוק את הנתון בסטטוס לפני שמחפשים באג.
  - **ריבוי חשבונות Google** אצל נופר גרם לתקלות (מסך ייבוא נכשל, /u/0/). לשים לב.
  - הורדת PDF כ-base64 בצד-לקוח = **html2pdf.js** (רסטר של הדף; עברית/RTL מושלמים כי זה צילום). מצב `.exporting` כופה פתיחת תתי-נושא + הסתרת פקדים.
